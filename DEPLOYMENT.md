# Инструкция по настройке автоматического деплоя

## Варианты настройки GitHub Pages

### Вариант 1: Сделать репозиторий публичным (Рекомендуется)

1. **Перейдите в настройки репозитория:**
   - Откройте ваш репозиторий на GitHub
   - Нажмите на вкладку "Settings"

2. **Сделайте репозиторий публичным:**
   - Прокрутите вниз до секции "Danger Zone"
   - Нажмите "Change repository visibility"
   - Выберите "Make public"
   - Подтвердите действие

3. **Активируйте GitHub Pages:**
   - В настройках перейдите в "Pages"
   - В разделе "Source" выберите "GitHub Actions"
   - Сохраните настройки

### Вариант 2: Использовать GitHub Actions (уже настроено)

Если вы хотите оставить репозиторий приватным, используйте GitHub Actions:

1. **Включите GitHub Pages в настройках:**
   - Settings → Pages
   - Source: "GitHub Actions"

2. **Workflow уже создан:**
   - Файл `.github/workflows/deploy.yml` настроен
   - Автоматический деплой при каждом push в main/master

## Что происходит после настройки

1. **Автоматический деплой:**
   - При каждом push в ветку main/master
   - GitHub Actions автоматически деплоит сайт
   - Время деплоя: ~2-3 минуты

2. **URL вашего сайта:**
   - `https://ваш-username.github.io/название-репозитория`
   - Например: `https://vasadasafasamaf.github.io/cursor-test`

3. **Мониторинг деплоя:**
   - Перейдите в "Actions" в вашем репозитории
   - Там вы увидите статус всех деплоев

## Структура файлов для деплоя

```
├── index.html          # Главная страница
├── styles.css          # Стили
├── script.js           # JavaScript
├── _config.yml         # Конфигурация Jekyll
├── .github/
│   └── workflows/
│       └── deploy.yml  # GitHub Actions workflow
└── .gitignore          # Исключения для Git
```

## Проверка работы

1. **После настройки:**
   - Сделайте любой commit и push
   - Перейдите в Actions → увидите запуск workflow
   - Через 2-3 минуты сайт будет доступен по URL

2. **Тестирование изменений:**
   - Внесите изменения в код
   - Commit и push
   - Сайт автоматически обновится

## Возможные проблемы

### GitHub Pages не активируется
- Убедитесь, что репозиторий публичный ИЛИ настроены GitHub Actions
- Проверьте, что в Settings → Pages выбран правильный источник

### Workflow не запускается
- Проверьте, что файл `.github/workflows/deploy.yml` находится в корне репозитория
- Убедитесь, что у вас есть права на запись в репозиторий

### Сайт не обновляется
- Проверьте логи в Actions
- Убедитесь, что все файлы закоммичены
- Попробуйте сделать пустой commit: `git commit --allow-empty -m "trigger deploy"`

## Дополнительные настройки

### Кастомный домен
1. Добавьте файл `CNAME` в корень репозитория
2. Укажите ваш домен в файле
3. Настройте DNS записи у вашего провайдера

### HTTPS
- GitHub Pages автоматически предоставляет HTTPS
- Работает для всех доменов

## Поддержка

Если возникли проблемы:
1. Проверьте логи в Actions
2. Убедитесь, что все файлы на месте
3. Попробуйте пересоздать workflow
